Signal

Fast, type-safe signals for Roblox.

VeSignal provides typed, priority-ordered events with connection pooling, multiple firing modes, and UDTF-derived type signatures. The Signature generic propagates to Fire, Wait, and all connect methods so your IDE catches argument type errors automatically.

Requires the new Luau type solver for full generic inference.

local Signal = require(ReplicatedStorage.VeSignal)

local onDamage = Signal.new() :: Signal.Signal<(victim: Player, amount: number) -> ()>

onDamage:Connect(function(victim, amount)
    print(victim.Name, "took", amount, "damage")
end)

onDamage:Fire(player, 25)

Functions​

new​

</>
Signal.new() → Signal<Signature>

Creates a new signal. Optionally annotate it with a type cast to lock in the Signature generic. 

local signal = Signal.new() :: Signal.Signal<(x: number) -> ()>

wrap​

</>
Signal.wrap(
rbxSignal: RBXScriptSignal--

The Roblox signal to wrap.

) → Signal<Signature>

Proxies a Roblox RBXScriptSignal into a VeSignal. All fires from the Roblox signal are forwarded through the wrapper. Destroying the wrapper also disconnects the underlying Roblox connection. 

local onTouch = Signal.wrap(workspace.Part.Touched)
onTouch:Connect(function(hit) ... end)

any​

</>
Signal.any(
...: Signal<Signature>--

Two or more signals to combine.

) → Signal<Signature>

Returns a new signal that fires whenever any of the given signals fires. The combined signal re-fires with the same arguments as the triggering input.

Destroy the combined signal to clean up all input connections.

local either = Signal.any(signalA, signalB)
either:Connect(function(...) ... end)
either:Destroy()

all​

</>
Signal.all(
...: Signal<Signature>--

Two or more signals to combine.

) → Signal<Signature>

Returns a new signal that fires once all given signals have fired at least once. Fires with the arguments of the last signal to complete the set. Subsequent fires from any input continue to fire the combined signal. 

local both = Signal.all(playerReady, mapReady)
both:Once(startGame)

Connect​

</>
Signal:Connect(
fn: Signature,--

The callback to invoke when the signal fires.

priority: number?--

Dispatch priority. Higher runs first. Default: 0.

) → Connection<Signature>

Registers a synchronous listener. Listeners with higher priority run first; the default priority is 0. Connections with the same priority fire in insertion order. 

local conn = signal:Connect(function(x) print(x) end)
conn:Disconnect()

ConnectSync is an alias for Connect.

ConnectAsync​

</>
Signal:ConnectAsync(
fn: Signature,--

The callback to invoke in a pooled thread.

priority: number?--

Dispatch priority. Default: 0.

) → Connection<Signature>

Registers a listener that runs in a pooled coroutine thread. The firing coroutine is not blocked and the listener may yield freely. 

signal:ConnectAsync(function(x)
    task.wait(1)
    doSomething(x)
end)

OnceTimeout​

</>

fn: Signature,--

The callback to invoke if the signal fires in time.

timeout: number,--

Seconds to wait before silently disconnecting.

priority: number?--

Dispatch priority. Default: 0.

) → Connection<Signature>

Registers a once listener with a deadline. If the signal fires before timeout seconds, fn is called and the connection is removed. If the timeout expires first, the connection is silently removed and fn is not called.

    openDoor()

Fire​

</>
Signal:Fire(
...: any--

Arguments forwarded to every listener.

) → ()

Fires the signal. Sync listeners are called directly on the current coroutine; async listeners (IsAsync = true) are dispatched to pooled threads. This is the recommended default firing method.

Has a fast path when AsyncCount == 0 — skips all thread machinery.

FireSync​

</>
Signal:FireSync(
...: any--

Arguments forwarded to every listener.

) → ()

Calls all listeners synchronously regardless of their IsAsync flag. The fastest firing mode. Use when all listeners are sync and re-entrance is not expected.

FireAsync​

</>
Signal:FireAsync(
...: any--

Arguments forwarded to every listener.

) → ()

Dispatches every listener to a pooled coroutine thread. Guarantees the firing coroutine is never blocked, even by sync-registered listeners.

FireDeferred​

</>
Signal:FireDeferred(
...: any--

Arguments forwarded to every listener.

) → ()

Schedules a Signal:FireSync on the next task.defer cycle. Use to break re-entrance or to guarantee the fire happens after the current frame.

FireSafe​

</>
Signal:FireSafe(
...: any--

Arguments deep-copied and forwarded to every listener.

) → ()

Fires the signal with per-listener pcall protection and deep-copied table arguments. Errors are surfaced as warn output rather than propagating. Use when listeners are untrusted or arguments must not be mutated across listeners.

Wait​

</>
Signal:Wait(
timeout: number,--

Maximum seconds to wait. Pass 0 or omit for no timeout.

priority: number?--

Dispatch priority for the internal listener. Default: 0.

) → ...any--

The arguments the signal was fired with, or nothing on timeout.

Yields the current coroutine until the signal fires, then returns the fire arguments. Must be called from inside a coroutine or task context.

If timeout > 0 and the signal does not fire in time, the coroutine resumes with no return values. 

local winner = roundEnded:Wait(60)
if winner then
    displayWinner(winner)
else
    displayDraw()
end

WaitPriority​

</>
Signal:WaitPriority(
priority: number?--

Dispatch priority. Default: 0.

) → ...any--

The arguments the signal was fired with.

Yields until the signal fires. No timeout. The optional priority controls where the internal listener is inserted in the queue. Must be called from inside a coroutine or task context.

GetListenerCount​

</>
Signal:GetListenerCount() → number

Returns the number of currently active connections.

HasListeners​

</>
Signal:HasListeners() → boolean

Returns true if at least one connection is active.

DisconnectAll​

</>
Signal:DisconnectAll() → ()

Disconnects every active connection and returns all connection objects to the internal pool. The signal remains usable after this call.

Destroy​

</>
Signal:Destroy() → ()

Disconnects all listeners, disconnects any proxied RBXScriptSignal, and clears the signal table. Do not use the signal after calling this.
